**🚀 이번 세션은 지시대로 push까지 완료했습니다** — 배포본(Vercel)에
XSS가 살아있던 상태였기 때문에 커밋 즉시 push, 보류하지 않음.

**커밋①(XSS 핫픽스)** — `state.profile.milestones[].label`이
`js/nav.js:61`(대시보드 타임라인, 로그인마다 자동 렌더)·`js/ai.js:26`
(AI 채팅 칩)에서 `esc()` 없이 `innerHTML`에 삽입되던 것 수정.
`${s.t}`→`${esc(s.t)}`, `${s.d}`→`${esc(s.d)}`,
`${dday(s.date)}`→`${esc(dday(s.date))}`(nav.js), `${c}`→`${esc(c)}`
(ai.js). 두 파일 전체 재검토 — 이 4곳 외 esc() 빠진 사용자 입력
보간 없음 확인(`p.maxArea`는 `+value` 강제로 항상 숫자라 애초에
안전, 그대로 둠). 새 로직 없이 기존 `esc()` 감싸기만.

**커밋②(매물 메모 WYSIWYG)** — `js/properties.js:774`
`esc(p.memo)`→`renderMd(p.memo)`(B-64에서 검증 완료된 marked.js+
DOMPurify 경로 재사용). 컨테이너에 `sc-md-content` 클래스 추가(다른
모든 renderMd() 표시 영역과 동일 패턴, 마크다운 요소별 스타일 적용을
위한 필수 동반 변경 — 신규 CSS 없음). `CLAUDE.md`의 renderMd 설명도
B-64 실측 결과에 맞게 갱신(같은 커밋).

- **검증**(Playwright, 두 커밋 모두):
  - 마일스톤 라벨에 `<img src=x onerror=...>` → 대시보드·AI 채팅
    양쪽에서 **실행 안 됨**, 이스케이프된 텍스트로만 노출(B-64 때
    실행 확인했던 것과 동일 payload로 재현→방어 확인). 정상 라벨
    ("결혼식")·날짜 표시 무회귀.
  - 매물 메모에 `**굵게**`+목록 입력 → `<strong>`·`<li>` 정상 렌더
    확인(수정 전엔 별표·대시 그대로 노출). 같은 경로에 `<script>`+
    `<img onerror>` 재주입 → 둘 다 무력화 재확인(회귀 없음). 빈
    메모도 크래시 없음.
  - `node --check js/nav.js`·`js/ai.js`·`js/properties.js` 전부 통과.
  - 로컬 python 테스트 서버·Playwright 스크립트는 레포 바깥
    scratchpad에서만 실행, 세션 종료 전 전부 삭제.
