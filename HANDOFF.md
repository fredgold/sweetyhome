# HANDOFF — B-46 매물탭 하단 여백/잔여 스크롤 제거 (2026-07-13)

## 1. 목표
`BACKLOG.md`의 B-46 처리. B-49 검증 중 발견한 매물탭 잔여 스크롤(~98px)의 원인을
특정 — 전역 컨테이너 `.wrap`의 `padding-bottom:64px`가 과함. 이를 축소하고, 뷰포트
고정 레이아웃인 데스크톱 매물탭에서는 조건부로 0까지 줄이는 것이 목표.

## 2. 완료
**커밋·push 완료.**

```
c58d70e fix: B-46 매물탭 하단 여백/잔여 스크롤 제거 (.wrap padding-bottom)
```

1. `.wrap`의 기본(무조건 적용) `padding-bottom`을 64px → 24px로 전역 축소.
2. 데스크톱 매물탭(`#panel-props.on`) 활성 시엔 `body:has(#panel-props.on) .wrap{
   padding-bottom:0}`로 조건부 0 적용 — `:has()` 셀렉터 사용(프로젝트가 이미 모던
   브라우저만 대상이라 지장 없음). `@media(max-width:760px)`의 별도 모바일 `.wrap`
   패딩 오버라이드(14/12/60)는 손대지 않음.

### 검증 방법
`node --check`(JS 변경 없음)/CSS 중괄호 균형/`git diff --check` 통과. Playwright로
데스크톱(1400×900):
- 매물탭에 가짜 단지 15건 주입 후 `.wrap`의 computed `padding-bottom`이 `0px`,
  잔여 스크롤이 B-49 시점 ~98px → 34px로 감소(완전 0은 아니나 대폭 개선).
- 대시보드·자산·액션·수집함 4개 탭 전환 시 `.wrap`의 `padding-bottom`이 여전히
  `24px`(0으로 안 새어나감), 각 탭 풀페이지 스크린샷으로 마지막 요소가 브라우저
  하단에 딱 붙지 않고 적당한 여백을 유지함을 시각 확인 — 무회귀.
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거, 테스트
  스크립트·임시 서버·스크린샷은 세션 종료 전 삭제.
