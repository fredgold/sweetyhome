`renderComplexes()`(`js/properties.js`)의 카드 템플릿에서 `weeklyBtn` 변수·렌더를

줄 조건을 `weeklyBadge` 하나로 단순화해 버튼 제거 후 빈 줄이 안 남게 함. 카드용
delegated 클릭 핸들러(`[data-weeklycheck]` 분기)도 함께 정리했지만, 공유 함수
`weeklyCheckComplex()`는 상세 모달 버튼이 계속 쓰므로 그대로 보존.

### 검증 방법
`node --check`/`git diff --check` 통과. Playwright로 데스크톱(1400×900)·모바일
(390×844) 양쪽에서 단지 2건(대표매물 `lastCheckedAt` 10일 전=미확인 뱃지 대상 /
방금 확인=뱃지 없음) 주입 후:
- 카드에 `[data-weeklycheck]` 버튼이 전혀 없음 확인.
- 미확인 카드엔 "7일+ 미확인" 뱃지 + 최근확인 날짜 유지, 확인 카드엔 뱃지 없이
  날짜만 — 스크린샷으로 빈 줄·여백 없음 시각 확인.
- 단지 상세 모달의 `cxDetailWeeklyCheckBtn` 클릭 → `lastCheckedAt` 즉시 갱신,
  상세 뱃지 사라짐 확인.
- 상세에서 확인 처리한 직후 카드 쪽도 뱃지가 사라지고 날짜가 갱신됨 — 카드·상세가
  같은 데이터 파이프라인을 공유함을 재확인(회귀 없음).
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거, 테스트
  스크립트·임시 서버·스크린샷은 세션 종료 전 삭제.
