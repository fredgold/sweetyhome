**커밋①(B-68)** — "기존 매물을 단지로 정리" 진입 버튼 2곳(`migStartBtn`,

**커밋②(B-19확)** — 단지 매칭이 (단지명+지역) 완전일치라 표기만
달라도("래미안 원베일리" vs "래미안원베일리(아파트)") 중복 단지가
생기던 문제. 완전일치 실패 시에만 퍼지 후보를 찾는
`findComplexCandidates()` 신설(①이름 유사: 괄호·"아파트" 접미사·공백
제거 후 비교 ②좌표 300m 이내, 기존 `haversineM()` 재사용) — 후보가
있으면 "이 단지에 추가할까요?" 제안 모달을 띄워 **사용자가 직접
선택해야만** 배정(자동 병합·자동 배정 없음). `properties.js` 외 무접촉
제약이라 모달은 JS로 `document.body`에 동적 주입, 새 CSS 없이 기존
`.modal`/`.mhead`/`.mbody`/`.mfoot` 재사용. `saveAsComplexListing()`
(수동 추가)·`calcCxImportStatus()`/`renderImportPreview()`(TSV 임포트)
양쪽 적용 — 임포트 쪽은 미리보기 표에 "비슷한 단지 있음" 배지+선택
드롭다운, 기본값은 "새 단지로 만들기"(자동 병합 아님).

**B-20 해소 여부: 해소 안 됨.** B-20은 "같은 단지에 동일 매물을 또
추가해도 안 걸러짐" 문제인데, 이번 건 "어느 단지에 배정할지"만
다뤄서 완전일치로 단지를 바로 찾는 경우(B-20이 실제 발생하는 시나리오)
엔 제안 로직 자체가 실행 안 됨. 실제 해소하려면 `saveAsComplexListing()`
에 `listingExists()`(임포트 경로가 이미 씀) 호출을 추가해야 하는데
이번 지시서엔 없어 손대지 않음 — **B-20은 별도 지시 필요**.

- **검증**(Playwright, 데스크톱 1280px+모바일 390px, 시나리오별 독립
  페이지로 상태 오염 방지):

    취소 → 아무것도 안 저장되고 폼 입력값 유지, "새 단지로 만들기" →
    명시적 선택 시 신규 생성, 좌표만 근접(다른 이름) → "위치 근접"

  - 임포트 경로: 드롭다운 노출·배지 표시, 후보 선택 시 기존 단지에
