### 버그 발견·회피 (중요 — 다음에 참고할 것)
`nav.js`의 기존 `dday()`(마일스톤 D-day 계산용)를 재사용하려 했으나,
Playwright 경계값 테스트로 "어제 마감·미완료" 액션이 잘못 "안 지남"으로
표시되는 버그를 발견. 원인: `dday()`가 `"YYYY-MM-DD"`를 `new Date(t)`로
파싱하는데, 날짜-only ISO 문자열은 **UTC 자정**으로 해석되는 JS 스펙상
함정이 있고, "오늘"은 로컬 `new Date()`를 쓰므로 KST(UTC+9)에서 매번 9시간
시차가 발생해 날짜 경계에서 하루가 밀림. `nav.js` 무접촉 원칙 때문에
그 함수는 안 고치고, `actions.js` 안에 `actDaysUntilDue()`를 새로 작성 —
`"YYYY-MM-DD"`를 연·월·일로 분해해 3-인자 `Date` 생성자(항상 로컬 시간으로
확정 해석)로 조립해 이 기능만 정확하게 함. **`dday()` 자체가 다른 곳
(마일스톤)에서 같은 함정을 갖고 있을 가능성은 이번 범위 밖이라 손 안 댐
— 필요하면 별도 항목으로 검토.**

### 검증 방법
`node --check`/CSS 중괄호 균형/`index.html` div 개폐 균형/`git diff --check`
통과. Playwright로:
- **applyGuards 라운드트립**: 필드 없는 기존 액션 → `''` 보정, 무손실.
- **담당 select**: 게스트 모드에서 옵션이 `OWNERS`와 정확히 일치.
- **추가/인라인 편집**: 담당·마감 입력·수정이 `state.actions`에 정확히 반영.
- **마감 경과 강조**(타임존 버그 수정 후 재검증): 어제 마감·미완료 → 빨간
  "D+1", 같은 조건이지만 완료된 액션은 강조 없음, 마감 미입력은 뱃지 자체
  없음.
- **정렬 무회귀**: `priority` 낮은 액션이 마감이 훨씬 지났어도 여전히
  `priority` 순서만 따름(마감 기준 재정렬 없음) — `#act_list .actrow`로
  정확히 스코프해 확인(대시보드 `renderTop3()`도 같은 `.actrow` 클래스를
  써서 전역 셀렉터가 섞이던 테스트 스크립트 버그를 도중에 바로잡음).
- 스크린샷으로 빨간 뱃지(미완료)와 무채색 뱃지(완료)가 명확히 구분됨을
  시각 확인.
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거,
  테스트 스크립트·임시 서버·스크린샷은 세션 종료 전 삭제.
