# HANDOFF — B-12 A안 재작업: 모바일 매물탭 풀스크린 지도뷰 (2026-07-12)

## 1. 목표
직전 세션에서 push한 B-12 B안(지도 상단 sticky + 세로 흐름 카드 스트립, `b9b7926`)이 "세로
스크롤 중 sticky 지도와 z-index가 섞여 깨진 느낌"이라는 피드백을 받아, 시안대로 A안(뷰포트
고정 풀스크린 지도 + 떠있는 카드 오버레이)으로 레이아웃을 교체. 마커연동·정렬·현위치·바텀시트
로직은 전부 재사용하고 레이아웃(지도 크기·카드 배치·부가 UI 위치)만 변경. 데스크톱(≥900px)은
절대 무변경이 제약 조건.

## 2. 완료
**A1~A4 전 단계 구현·검증·커밋 완료. push는 아직 안 함(사용자 명시 요청 대기).**

### 단계별 요약
- **1/4**: `#panel-props`를 `position:fixed`(nav 아래 뷰포트 전체, `overflow:hidden`)로 전환 — 페이지 세로 스크롤 제거가 핵심(sticky+z-index 혼선의 근본 원인). `.mapcard`/`#overviewMap`이 이를 기준으로 절대위치 풀스크린. 이 단계에서는 카드·칩·FAB·검색·필터·폼·레거시를 전부 비움(다음 단계에서 순차 재노출).
- **2/4**: `#complexSection`(카드 스트립)을 지도 하단 오버레이로, `.cx-sort-chips`를 좌상단 오버레이로 재노출 + z-index 체계 확립(지도0 < 오버레이10~20 < 기존 `.status-picker`(2000)/`.modal`(1000+) 인프라는 무변경, 상대순서만 맞춤).
- **3/4**: 검색·필터·내보내기·시트붙여넣기·임장루트를 지도 우상단 "⋯" 메뉴로 통합(`showMoreMenu()` 확장). 현위치+매물추가(신규 FAB)를 우하단에 세로로 쌓음. 매물 추가 폼을 지도를 밀어내지 않는 하단시트로 전환.
- **4/4**: 실데이터(단지 3개)로 Playwright 390px/1400px 전체 회귀 검증.

### 검증 중 발견·수정한 버그 3건 (전부 이번 세션 내 발견 즉시 수정)
1. **`--nav-h` 헤더 높이 누락** (2단계에서 발견): `--nav-h`는 `.apptabs`(탭바) 자체 높이만 담고 `header`(로고+타이틀) 높이를 빠뜨림 → `#panel-props{top:var(--nav-h)}`가 헤더 하단부와 겹쳐 정렬칩이 탭바 위에 그려짐. `header`+`.apptabs` 합산값을 담는 `--topbar-h`를 새로 추가(`updateNavHeightVar()` 확장 + `initOverview()` 진입마다 재측정)해 교체. 기존 `--nav-h` 용례(다른 sticky 오프셋)는 무변경.
2. **⋯메뉴 폭 오버플로우** (4단계에서 발견): 검색+필터가 메뉴에 추가되며 실제 폭이 옛 가정(230px)보다 커짐(필터바 6행 포함 시 최대 366px) → `.ph-more-menu`에 `max-width` 없어 메뉴가 뷰포트 밖으로 넓어짐(가로 스크롤 발생). `max-width:calc(100vw - 24px)` + 내부 `#unisearch`/`#cxFilterBar` `width:100%`로 고정.
3. **⋯메뉴 좌측 위치 계산 하드코딩** (4단계에서 발견): `showMoreMenu()`의 좌측 클램프가 옛 3항목 메뉴 폭(230px)을 가정 → 확장된 메뉴에서 우측이 뷰포트 밖으로 튀어나감. `menu.offsetWidth` 실측 기반으로 교체(데스크톱 `<480px` 3항목 메뉴는 영향 없음, 오히려 더 정확해짐).

### 검증 방법
매 단계: `node --check` 전체 JS + CSS 중괄호 균형 + `git diff --check` 통과 후 다음 단계 진행.
Playwright(임시 설치, `npm uninstall`로 흔적 제거)로 정적 서버(UTF-8 mis-serve 있는 `python -m
http.server` 대신 커스텀 Node 서버) + 게스트모드(`#loginGuest`)로 매 단계 390px/1400px 확인.
4단계에서는 `state.complexes`/`state.listings`에 실제 단지 3개를 직접 seed해 카드↔마커
연동(`focusCxCard`), 카드 탭→바텀시트(`openComplexDetail`), ⋯메뉴 실사용까지 확인.

## 3. 미완
- **실기기 모바일 배포 검증 대기.** 로컬 PIN·네이버 지도 도메인 제약(Client ID가 배포 도메인에만
  허용)으로 이번 세션도 정적 서버+목데이터로만 확인. 실제 배포 도메인+실기기(iOS/Android
  브라우저)에서: 카드 좌우 스와이프 스냅 체감, FAB 터치 영역 크기 적정성, ⋯메뉴 터치 후 필터
  칩 조작감, 바텀시트/폼시트 오픈 애니메이션 체감 확인 필요.
- **push 안 함** — 사용자가 이번 요청에서 push를 명시하지 않음. 다음 지시("push하자" 등) 대기.
- `.mapcard.expanded`/`.mapcard.collapsed`(구 "크게 보기/접기" 토글) CSS는 이제 모바일에서
  완전히 도달 불가능한 죽은 코드가 됨(헤더 자체가 모바일에서 숨겨짐 + 지도가 항상 풀스크린) —
  명시적 요청 없어 정리하지 않음. 필요 시 별도 지시로 정리 대상.

  자체의 `initOverview()`(어차피 `switchPanel('props')`가 호출)에 넣어 파일 충돌 리스크 회피.

  `package.json`엔 흔적 없음.
