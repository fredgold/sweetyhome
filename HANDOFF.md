# HANDOFF — B-51 대형 화면(외부 모니터) 매물탭 전체폭 활용 (2026-07-13)

## 1. 목표
`BACKLOG.md`의 B-51 처리(B-50과 "매물탭 화면 활용 마무리" 그룹으로 이어지는
작업). `.wrap`의 `max-width:1440px` 제한 때문에 큰 모니터(1920/2560/4K)에서
양옆에 큰 여백이 남고, 특히 매물탭 지도가 넓을수록 좋은데도 1440px에 갇혀
있던 문제를 CSS로 해결하는 것이 목표. 문서형 탭(대시/자산/액션/수집함)은
가독성 위해 현재 폭 유지.

## 2. 완료
**커밋·push 완료.**

```
093a67e feat: B-51 대형 화면(외부 모니터) 매물탭 전체폭 활용
```

`body:has(#panel-props.on) .wrap{max-width:none;}` 한 줄 추가 — B-46이 이미
검증해둔 같은 셀렉터 패턴(`body:has(#panel-props.on) .wrap{padding-bottom:0}`)을
재사용해 매물탭 활성 시에만 스코프. `.grid`의 사이드바 트랙은 고정 폭
(`minmax(0,var(--sidebar-w,360px))`)이라 영향 없고, 지도 트랙
(`minmax(0,1fr)`)이 늘어난 폭을 구조적으로 그대로 흡수 — grid 자체는 변경
없음.

### 검증 방법
`node --check`(JS 변경 없음)/CSS 중괄호 균형/`git diff --check` 통과. Playwright로
1400px·1920px·2560px 세 뷰포트에서 매물탭·대시보드 탭을 각각 측정:

| 뷰포트 | 매물탭 `.wrap` 폭 | 매물탭 지도 폭 | 대시보드 `.wrap` 폭 |
|---|---|---|---|
| 1400px | 1400px | 976px | 1400px |
| 1920px | 1920px(전체폭) | 1496px | **1440px(고정)** |
| 2560px | 2560px(전체폭) | 2136px | **1440px(고정)** |

- 사이드바 컬럼 폭은 세 뷰포트 모두 360px로 불변.
- 1400px 이하는 `max-width` 자체가 원래도 제약 요인이 아니라 수정 전/후
  완전히 동일 — 무회귀 확인.
- 1920px 스크린샷으로 매물탭은 지도가 화면 대부분을 채우고, 대시보드는
  여전히 가운데 정렬된 1440px 폭으로 편안하게 읽히는 대비를 시각 확인.
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거,
  테스트 스크립트·임시 서버·스크린샷은 세션 종료 전 삭제.
