- **수정 파일**: `js/nav.js` 단 1개, `dday(t)` 함수만(12줄 추가/1줄 삭제).
  시그니처·반환 형식(`'D-n'`/`'D+n'`) 불변.
- **수정 방식**: 정규식(`/^(\d{4})-(\d{2})-(\d{2})/`)으로 연·월·일 추출 →
  3-인자 `Date` 생성자(로컬 자정 확정 해석)로 재조립. 매칭 실패 시 기존

- **검증**(Playwright, 로컬 정적 서버+게스트모드, 오늘 2026-07-13 기준):
  - `dday('2026-07-14')`(내일) = `'D-1'` ✓
  - `dday('2026-07-13')`(오늘) = `'D-0'` ✓
  - `dday('2026-07-12')`(어제, **이전엔 버그로 `'D-0'`였던 핵심 케이스**) =
    `'D+1'` ✓ — 버그 재현·수정 확인 완료.
  - 형식 오류 입력(`'not-a-date'`, `''`, `undefined`)은 기존과 동일하게
    `'D+NaN'`로 안전 폴백(크래시 없음, 회귀 아님).
  - 대시보드 마일스톤 타임라인(`renderJourney()`)에서 "D-172", "D-537" 등
    정상 값 렌더링 스크린샷으로 육안 확인.
  - `node --check js/nav.js`·`git diff --check` 통과.
- **부수 효과(긍정적)**: `dday()`를 쓰는 다른 모든 곳(마일스톤 D-day 표시 등)이
