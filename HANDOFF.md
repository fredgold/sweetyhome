### 검토했지만 보류한 것 — 헤더+탭바 같은 줄 배치
지시문의 "가능하면 같은 줄 배치, 마크업 재구성 필요하면 세로 압축까지만"
조건에 따라 검토. `header`와 `.apptabs`(`<nav>`, 독립적으로 `position:sticky`
사용)는 `.wrap`의 별개 형제 요소라 마크업 변경 없이 CSS만으로 자연스럽게
한 줄로 합치기 어렵다고 판단해 보류 — 세로 압축만 진행. **후속으로 마크업
재구성이 필요한 "같은 줄 배치" 과제가 남아있음**(별도 항목 검토 필요, 이
작업은 nav.js에도 손대야 할 가능성이 높아 Codex와 파일 락 조율 필요).

### 검증 방법
`node --check`(JS 변경 없음)/CSS 중괄호 균형/`git diff --check` 통과. Playwright로
데스크톱(1400×900)에서 `git stash`로 수정 전/후 직접 비교(가짜 단지 15건 주입,
동일 조건):
- **정량 측정**: `--topbar-h` 151.75px→132.5px(**-19.25px**). 매물탭 지도
  높이 665.25px→684.5px(**+19.25px**, B-49의 flex 레이아웃이 압축분을 그대로
  흡수). 리스트 내부 스크롤 영역(`#complexSection.clientHeight`)
  447→466px(**+19px**).
- 대시보드·자산·매물 탭 스크린샷으로 헤더 압축 후에도 로고·제목·버튼이 읽기
  편하고 안 깨짐, 매물탭에서 카드가 이전보다 더 많이 보임을 시각 확인.
- 모바일(390×844)에서 `header margin-bottom`(10px)·`.mark width`(34px)가
  기존 760px 브레이크포인트 값 그대로임을 computed style로 확인 — 데스크톱
  전용 규칙 무누출, 완전 무회귀.
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거,
  테스트 스크립트·임시 서버·스크린샷은 세션 종료 전 삭제.
