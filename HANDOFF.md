# HANDOFF — B-39 후보 즐겨찾기 토글(상단 고정+필터) (2026-07-13)

## 1. 목표
`BACKLOG.md`의 B-39 처리. 후보 단지 중 "지금 집중해서 보는 것"을 즐겨찾기
(핀)로 표시하고 상단에 띄움. `complexStatus`가 이미 우선순위 축을 거치므로
별점·순위·등급은 만들지 않고 boolean 하나로 축소. "기록·관리 중심 CRM(P0)"
그룹의 마지막 항목(B-28→B-18→B-38→**B-39**→B-37).

## 2. 완료
**커밋·push 완료.**

```
d943fd5 feat: B-39 후보 즐겨찾기 토글(상단 고정+필터)
```

- **스키마**: `complexes[].favorite`(boolean, 기본 `false`) 신설.
  `applyGuards()` + 단지 생성부 3곳 반영.
- **토글 UI**: 카드(`c-badge-col`)와 단지 상세 모달 헤더에 별 버튼. 공용
  `toggleFavorite(id)` 하나로 양쪽 처리(새 저장경로 없음). 아이콘은 기존
  `ICSVG.star` 재사용, `.ic` 기본값이 이미 "off=외곽선"이라 on 상태
  (골드 `--line9` 채움) CSS만 3줄 추가.
- **버블링 분리**: 카드 전체가 `data-cxopen` 클릭 타깃이라, 기존
  `.c-routecheck-wrap`이 쓰던 "delegated 핸들러에서 먼저 걸러 return"
  패턴을 재사용해 `stopPropagation()` 없이 상세 진입과 분리.
- **정렬**: `favoritesFirst()` 신설 — `Array.sort`의 안정 정렬(ES2019+)을
  이용해 기존 정렬 위에 즐겨찾기만 최상단으로 얹음(그룹 내 기존 순서는
  유지).
- **필터**: boolean이라 드롭다운 없는 단순 on/off 토글 칩으로
  `cxFilterBar`에 추가, 기존 트리거 활성 스타일 재사용.

### 조사 중 발견한 것
데스크톱은 `cxSort`(최신순/가격순/거리순) 자체가 애초에 적용 안 되고
`state.complexes`의 자연 배열 순서를 그대로 쓰고 있었음(기존 동작, 이번
세션에서 새로 발견). 즐겨찾기를 데스크톱에도 상단 고정하려면
`favoritesFirst()`를 그 경로에도 별도로 적용해야 했음 — 데스크톱의 기존
"정렬 없음" 특성 자체는 건드리지 않고 즐겨찾기만 위에 얹음.

### 검증 방법
`node --check`/CSS 중괄호 균형/`index.html` div 개폐 균형/`git diff --check`
통과. Playwright로:
- **applyGuards 라운드트립**: `favorite` 없는 기존 단지 → `false` 보정,
  B-38 `verdict` 등 기존 필드 무손실.
- **버블링 분리 실측**: 카드 별 버튼 클릭 → 상세 모달 안 열림 + `favorite`는
  정상 토글됨.
- **정렬 실측**: 3개 단지 중 마지막(자연순서 기준)을 즐겨찾기 → 카드 순서가
  `[즐겨찾기한 것, 나머지는 기존 순서 유지]`로 바뀜 확인.
- **토글→저장→재로드**: `applyGuards()` 재로드 시뮬레이션에도 값 유지.
- **카드↔상세 동기화**: 상세에서 토글 → 카드 버튼도 즉시 갱신됨.
- **"즐겨찾기만" 필터**: 필터 클릭 → 즐겨찾기만 남고 필터 버튼 활성 표시.
- 스크린샷으로 채워진 골드 별 아이콘 + 활성 필터 칩 시각 확인.
- 테스트 중 불완전한 픽스처로 `applyGuards()`를 연속 호출하는 스트레스
  시나리오에서 산발적 에러가 관찰됐으나, 단일 사이클 재현 테스트로는 전혀
  발생하지 않아 테스트 하네스 아티팩트로 판단(이번 세션에서 반복 확인된
  패턴 — 실제 사용자 흐름은 `applyGuards()`를 1회만 호출).
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거,
  테스트 스크립트·임시 서버·스크린샷은 세션 종료 전 삭제.
