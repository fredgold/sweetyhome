### B-35 요약

구조로 교체. `cxFilters` 상태·`cxMatchesFilters()`·옵션 렌더 로직은 전혀 안 건드리고
UI만 재구성(요구사항대로). 탐색그룹(`regionGroup`)은 생성부 3곳에서 항상 `''`로

드롭다운은 `showMoreMenu()`와 동일한 "열릴 때만 body로 이동, 닫으면 원복" 패턴
(`openCxFilterDropdown`/`closeCxFilterDropdowns`, `js/properties.js`)으로 구현 —
처음엔 `.cxf-group` 안에 `position:absolute`로 뒀다가 `#cxFilterBar{overflow-x:auto}`가
CSS 스펙상 `overflow-y`까지 강제로 `auto`가 돼 드롭다운이 통째로 잘려 안 보이는 버그를
Playwright 스크린샷으로 발견 → body 이동 패턴으로 해결.

모바일: 더보기(⋯)메뉴에서 필터 제거, 지도뷰 상단 오버레이 2번째 줄(정렬칩 아래)에 상시
노출. 리스트뷰에선 `syncListToolbar()` 이동 목록에 편입돼 `#cxListToolbar` 둘째 줄로
들어감(sticky 불투명 바 유지).

### 검증 방법
`node --check` 전체 + CSS 중괄호 균형 + `git diff --check` 통과. Playwright로 로컬 정적
서버(임시 node 서버, UTF-8 charset 명시 — `python -m http.server`는 인코딩이 깨져 사용
못 함) + 게스트모드 + **가짜 단지 3건을 `state.complexes`/`state.listings`에 직접 주입**해
검증(실제 데이터 없이는 필터 UI가 빈 상태로 숨겨져 있어 이 방식이 필요했음):
- 데스크톱(1400px)·모바일(390px) 양쪽에서 트리거 라벨이 선택값 반영(`면적: 59㎡ 이하`),
  카드 실제 필터링(3→1건) 확인.
- 드롭다운이 뷰포트 안에 완전히 들어옴(마지막 그룹 포함, clamp 로직 확인) 확인.
- 모바일 지도뷰: 필터 바가 정렬칩 줄과 안 겹침(바운딩박스 실측: sortBox bottom≈200.5,
  filterBox top=216.5).
- 모바일 리스트뷰: 필터 바가 sticky 툴바 둘째 줄로 들어가고 카드와 안 겹침.
- ⋯메뉴에 필터가 더 이상 없음(검색·내보내기·시트붙여넣기·임장루트만) 확인.
- 데스크톱 무회귀, XSS 무접점(칩 라벨 `esc()` 통과).
- 로컬 테스트 도메인이 네이버 지도 API 키 허용 도메인이 아니라 지도 자체는 "인증 실패"로
  표시 — 배포 환경과 무관한 로컬 한계, 필터 바 레이아웃 검증에는 영향 없음.
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거(`package.json`
  흔적 없음), 테스트 스크립트·임시 서버·스크린샷은 전부 세션 종료 전 삭제.
