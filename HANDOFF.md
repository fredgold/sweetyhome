# HANDOFF — B-44② 단지 카드 밀도 개선 + 대표가격 헤드라인급 강조 (2026-07-13)

## 1. 목표
`BACKLOG.md`의 B-44②(B-44①은 이전 세션에서 완료) 처리. 대표매물 보증금이 다른
칩들과 똑같은 크기라 눈에 안 띄던 것을 단지명 바로 아래 헤드라인급으로 강조하고
("사람은 단지가 아니라 가격을 기억"), 카드 세로 여백을 더 축소해 뷰포트에 보이는
단지 수를 늘리는 것이 목표.

## 2. 완료
**커밋·push 완료.**

```
ac88572 feat: B-44② 단지 카드 밀도 개선 + 대표가격 헤드라인급 강조
```

1. **대표가격 헤드라인**: `c-head-text`에 `c-price`(16px, 굵게, 골드
   `--line9-deep`) 신설, 단지명(`c-headline`)과 위치정보(`c-sub`) 사이에 배치.
   `c-meta`의 중복 deposit 칩과 이제 안 쓰는 `.chip.deposit` CSS는 제거.
2. **카드 밀도**: `#complexSection` 스코프로만 데스크톱 전용 압축(`card
   margin-bottom` 12→4px, `c-top padding` 14→8px, `c-meta margin-bottom`
   8→3px). 카드 하단 패딩은 기존 인라인 style(고정값이라 조건부 축소 불가)을
   `cx-card-body` 클래스로 바꿔 데스크톱에서만 14→8px로 축소.

### 시행착오 (다음 유사 작업에 참고할 것)
처음 `c-price`를 19px로 뒀더니 새 가격 줄의 높이(~29.5px)가 패딩·마진
절감분(~19px)보다 커서 카드가 오히려 **커졌고**(143→157px) 뷰포트에 보이는
카드 수도 그대로였음(3→3) — Playwright로 `cardOuterHeight`/
`visibleCardsInViewport`를 실측하지 않았다면 놓쳤을 회귀. `c-price`를 16px로
줄이고 패딩 축소폭을 더 키워 최종 순감소(143→140px, 3→4개 카드)를 확보. **UI
밀도 작업은 "요소 추가 + 패딩 축소"를 동시에 할 때 반드시 순수 높이 변화를
실측해서 검증해야 함** — 시각적으로 "슬림해 보인다"와 실제 픽셀 수가 줄었다는
사실이 다를 수 있음.

### 검증 방법
`node --check`/CSS 중괄호 균형/`git diff --check` 통과. Playwright로 데스크톱
(1400×900)에 가짜 단지 15건 주입 후:
- 레거시 카드(`#legacyWrap .card`)의 spacing이 원래 값 그대로 — `#complexSection`
  스코프가 정확히 격리됨(레거시 `.rail .card`는 같은 클래스를 공유하지만 영향
  없음).
- `c-price` 텍스트·폰트 크기·색상 확인, `c-meta`에서 deposit 칩 제거 확인.
- 카드 margin-bottom·padding 축소값 확인, 뷰포트 안 보이는 카드 수 3→4 증가
  확인(스크린샷 시각 확인 포함).
- 모바일(390×844)에서 `c-price`가 동일 크기로 잘 보이되 `c-top padding`은
  기존 480px 브레이크포인트 값 그대로 — 데스크톱 전용 규칙 무누출, 터치
  타깃·가독성 무회귀.
- 임시 설치한 `playwright`는 `npm install --no-save` → `npm uninstall`로 제거,
  테스트 스크립트·임시 서버·스크린샷은 세션 종료 전 삭제.
