# AGENTS.md — 스위티홈 (SweetyHome)

> 이 파일은 이 레포에서 작업하는 모든 AI 코딩 에이전트(Codex, Claude Code 등)가 반드시 따르는 규칙이다.
> 프로젝트의 전체 맥락·현황·데이터 모델은 `PROJECT.md`가 원본(SSOT)이다. 작업 전 `PROJECT.md`를 먼저 읽어라.
> 이 파일과 `PROJECT.md`가 충돌하면 `PROJECT.md`를 따르고, 사람(테디)에게 알려라.

## 프로젝트 한 줄
테디 ♥ 연정의 결혼·서울 내집마련 준비 보드 웹앱. 단일 `index.html`(Vanilla JS) + `api/*.js` 서버리스, Vercel 배포.

## 불변 제약 (절대 어기지 말 것)

1. **단일 `index.html` 유지.** 프레임워크·번들러·빌드 도구 도입 금지. 모든 새 기능은 `index.html` 안에 넣는다. npm/패키지 매니저 도입 금지.
2. **저장은 하나의 상태 키 `sweetyhome`.** Upstash Redis(`/api/state`) 우선, localStorage 폴백. 기존 데이터 스키마를 깨지 마라. 새 필드는 반드시 DEFAULT 값과 load 가드를 추가해 기존 사용자 데이터가 안 깨지게 한다. (기존 매물/스크랩 데이터 보존이 최우선)
3. **AI 호출은 프런트의 `claudeAPI(messages, tools, system)` → `/api/messages` 프록시만 사용.** `api.anthropic.com` 직접 호출 금지. API 키·PIN·토큰을 코드에 하드코딩하는 것 절대 금지. 모든 시크릿은 환경변수로만.
4. **모델 기본은 Haiku.** 뉘앙스가 필요한 매물 리포트·상담에만 상위 모델 분리를 제안할 수 있으나, 제안 시 비용 증가를 반드시 함께 언급한다. 파싱·추출·자동화에는 AI를 쓰지 말고 결정론적 JS 로직을 우선한다.
5. **웹검색은 `tool = web_search`.** 이미지 입력은 base64 content 블록. 인스타그램 등 외부 사이트 자동 스크래핑은 불가 → 붙여넣기/스크린샷 방식으로 설계한다.
6. **디자인 토큰 재사용.** 색 역할: 골드=매물, 그린=자산, 보라=AI. 실제 값은 `index.html`에 정의돼 있으니 새로 만들지 말고 기존 토큰을 읽어 쓴다. 한국어 UI, 큰 버튼, 초보 친화.
7. **API 엔드포인트**: `/api/messages`, `/api/state`, `/api/geocode`, `/api/health`, `/api/login`, `/api/_auth`. 새 엔드포인트 추가는 신중히, 사람 확인 후.

## 배포
- `git push` → Vercel 자동 배포. 별도 빌드 없음.
- 새 환경변수가 필요하면 코드에 넣지 말고 "Vercel Settings에 이 env를 추가해 달라"고 사람에게 요청한다.

## 다중 에이전트 핸드오프 규칙 (Codex ↔ Claude Code)

이 레포는 Claude Code와 Codex가 함께 작업한다. 충돌을 막기 위해:

1. **`index.html`은 한 번에 한 에이전트만 편집한다.** 다른 에이전트가 이 파일을 편집 중일 수 있으면, 먼저 `git status`로 커밋 안 된 변경이 있는지 확인하고, 있으면 사람에게 물어라. 절대 커밋 안 된 남의 변경분 위에 덮어쓰지 마라.
2. **핸드오프는 커밋 단위로.** 작업을 마치면 즉시 의미 있는 메시지로 커밋한다. 큰 작업은 단계마다 자주 커밋해, 다른 에이전트가 언제든 깨끗한 지점에서 이어받을 수 있게 한다.
3. **멈출 때 상태를 남긴다.** 작업이 중간에 끊기면(토큰 소진 등), 마지막에 반드시 남긴다: "완료한 것 / 진행 중이던 것 / 다음에 할 것"을 3줄 이내로. 커밋 메시지나 `PROJECT.md`의 작업 로그에.
4. **큰 작업은 시작 전에 단계 체크리스트를 만든다.** 그래야 이어받는 에이전트가 어디까지 됐는지 파악한다.
5. **작업 완료 후 검증 포인트를 사람에게 알린다.** (예: "로컬에서 매물탭 열어 붙여넣기 되는지 확인해 주세요")

## 사람(테디)에 대해

- 큰 결정·스키마 변경·비용 발생은 마음대로 하지 말고 먼저 물어본다.
